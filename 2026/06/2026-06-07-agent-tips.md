# 🛠️ Agent Tips 2026-06-07

> 今日主题：Claude Code 的 "重试艺术" + Cursor 的 `.cursorrules` 小魔法

## 1. Claude Code：失败不是终点，是 new context 的起点

Claude Code 有个容易被忽略的隐藏 buff——**每次运行失败时，它其实已经悄悄把错误信息、堆栈、上下文都塞进对话历史里了**。这时候你别急着删了重来，直接说：

> "fix this" 或 "分析一下为什么报错"

它比你自己重新描述问题要准得多。因为 AI 看到的是完整的执行环境，而你口头描述往往只丢了 30% 的有效信息。这相当于 **"用错误本身当 prompt"**，省事儿还精准。

**Pro tip**: 如果报错太长，可以 `Ctrl+C` 中断后输入 `!` 查看历史命令，再针对性地让 Claude 去修。

## 2. Cursor：给项目装一个 "团队默契说明书"

Cursor 的 `.cursorrules` 文件（放在项目根目录）相当于给 AI 写了个**入职手册**。你可以在里面定义：

- 代码风格（比如 "用 Go 的标准错误处理，不要 panic"）
- 项目架构偏好（"所有业务逻辑放在 pkg/ 下，不要直接写在 main 里"）
- 甚至一些内部梗（"我们管这个叫 'router' 不叫 'handler'"）

最爽的是——**它跟着项目走，换了机器、换了人，AI 的表现是一致的**。比每次新开聊天窗口重新交代背景要稳得多。

```
# 示例 .cursorrules
- 用 TypeScript，strict mode 开满
- 所有 API 调用必须经过统一的 request 封装
- 不要给组件加默认 export，用命名导出
```
