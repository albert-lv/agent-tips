# Agent 暗号手册：两个让 AI 真正"懂你"的隐藏技巧

> 2026-06-09 | 周二早 ☀️

早上好，向上和小惠！今天分享两个很多老手都在用、但新手容易忽略的 Agent 调教技巧。

---

## 技巧一：给 Agent 一本「你的使用说明书」

Cursor 有 `.cursorrules`，Claude Code 有 `.claude.md`，Kimi Code 也支持项目级指令文件。在这些文件里写清楚你的偏好，Agent 每次对话都会自动带上。

**别写废话，写你真正的痛点：**

```markdown
- 我偏好英文变量名，但注释用中文
- 写 Python 时，类型提示必须加，不准用 Any
- 修改代码时，优先用 diff 模式，不要整段重写
- 遇到不确定的地方，先问我，不要擅自决定
```

**效果：** 你再也不用每次开场白都重复"请用中文注释"了。Agent 会自己翻这本说明书，越用越顺手。

---

## 技巧二：`@` 符号不是装饰，是精准定位器

很多人 `@` 个文件名就完了，其实可以 `@` 更细：

- `@file.py` → 引用整个文件（适合大上下文）
- `@file.py:42` → 定位到第42行（适合说"这里改一下"）
- `@ClassName` 或 `@function_name` → 跨文件引用符号（适合重构时）

**真实场景：** 你说"把那个登录逻辑改成用 JWT"，Agent 可能在三个文件里都有登录相关代码，瞎改一通。但如果你说"把 `@auth.py:LoginHandler` 改成 JWT"，它就知道精确目标了。

---

## 一句话总结

Agent 不是读心术大师，是执行力超强的实习生。你给它的指令越具体，它翻车越少；你给它一本说明书，它越用越像你的专属助手。

**试试今天给你的项目加一条 `.cursorrules` 或 `.claude.md`，明天就能感受到差别。**
