# 🧠 AI Agent 的「脑容量管理术」

> 日期：2026-06-20（周六）  
> 致：向上 & 小惠 —— 愿你们的 Agent 永远不忘事

---

## 今天聊一个隐形痛点：Agent 的「金鱼脑」

你们有没有遇到过这种情况——跟 Claude Code 或者 Kimi Code 聊了半天，项目背景交代得清清楚楚，结果它突然「失忆」了？

> *「等等，我们刚才在说哪个文件来着？」*

别急着骂它。这不是 bug，是**上下文窗口（Context Window）**的物理限制。再聪明的 Agent，脑容量也是有限的。今天分享两个超实用的「脑容量管理」技巧，让你的 Agent 从金鱼变海豚 🐬

---

## 技巧一：定期打「记忆结」（Checkpoint）

就像游戏里要存档一样，跟 Agent 长时间协作时，**每隔 15-20 轮对话，让它主动总结一次当前状态**。

**具体做法：**

```
"帮我总结一下我们目前的状态：
- 正在改哪个功能？
- 改了哪些文件？
- 还有什么待办？
- 把总结写进 PROJECT_STATE.md"
```

**为什么好用：**
- Agent 自己总结的，它下次读到记得最牢
- 写成文件后，即使开新会话也能快速恢复上下文
- 相当于给你们的对话按了「存档点」

💡 **进阶玩法**：让 Agent 把状态总结写成待办清单格式，下次直接说「从待办第 3 条继续」就能无缝衔接。

---

## 技巧二：大段内容走「文件通道」，别走「对话通道」

这是很多人踩过的坑：把几百行报错日志、整个配置文件的内容，直接贴进对话框让 Agent 分析。

**问题在哪？**
- 挤占了宝贵的上下文窗口
- Agent 可能只看得到后半截，前半截被「遗忘」了
- 下次新会话还要重新贴一遍

**正确姿势：**

| ❌ 别这样 | ✅ 这样做 |
|---------|---------|
| 把 200 行日志贴进对话框 | 写进 `debug_log.txt`，然后跟 Agent 说「分析一下这个文件」 |
| 直接粘贴整个 JSON 配置 | 保存为 `config.json`，让 Agent 自己读 |
| 口述项目结构 | 让 Agent 生成 `tree.md`，下次直接读文件 |

**一句口诀**：
> 📁 **能走文件的，不走对话；能引用路径的，不贴全文。**

---

## 实战组合拳 🥊

开场白模板（复制即用）：

```
我们先读一下 PROJECT_STATE.md 了解当前进度，
然后你来分析 src/ 下的代码结构，
最后我们聚焦解决 [具体问题]。
```

这样 Agent 一上来就有完整的背景信息，不需要你们重新交代一遍。省下的上下文额度，留给真正需要推理的部分。

---

## 一句话总结

**Agent 不是记不住，是容量有限。帮它管好记忆，它就能帮你管更大的项目。**

试试今天这两个技巧，下次开长会的时候你会感谢自己的 📌
