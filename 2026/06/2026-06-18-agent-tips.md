# 2026-06-18 Agent Tips

## 今日主题：「对话压缩术」—— 让 Agent 越聊越清醒

**一句话总结：** 长对话会让 Agent 变笨，学会"压缩"能让它一直保持满血状态。

### 技巧一：Claude Code 的 `/compact` —— 一键清缓存

和 Claude Code 聊到后面，它是不是开始"健忘"了？其实是因为它背了太多上下文，CPU 快烧了。

**试试这个：**
```
/compact
```

作用：把之前的对话历史压缩成一份摘要，清空缓存，但保留核心信息。等于给 Agent 吃了一粒"薄荷糖"——瞬间清醒。

> 💡 最佳时机：感觉 Agent 开始答非所问、重复建议，或者你开始新项目模块时。

### 技巧二：Kimi Code 的「断点续聊」—— 文件即上下文

Kimi Code 没有 `/compact` 那种魔法命令，但它有个更野的路子：**用文件当对话 backbone**。

操作步骤：
1. 项目根目录建一个 `AGENT_CONTEXT.md`
2. 把当前任务目标、关键决策、待办事项写进去
3. 每次开新会话时丢一句：`@AGENT_CONTEXT.md 帮我继续`

**好处：** 换会话不丢上下文，Agent 永远知道"咱们现在到哪了"。

### 对比速览

| 工具 | 清缓存方式 | 适合场景 |
|------|-----------|---------|
| Claude Code | `/compact` | 单会话长期作战 |
| Kimi Code | `AGENT_CONTEXT.md` | 多会话接力、团队协作 |
| Cursor | 点击 "New Chat" 但保留项目索引 | 模块切换、bug 修复 |

### 今日行动建议

今天写代码时，试试在 **30 轮对话后** 用一次 `/compact`（或等效操作），观察 Agent 的响应质量有没有回升。
