## 今日关键词：上下文压缩

有没有遇到过这种经典场面——跟 Claude Code 聊了一个小时，项目结构、业务逻辑、各种约定都铺垫好了，突然开始答非所问，或者干脆开始重复之前说过的内容？

大概率不是 Claude 变傻了，是**上下文窗口满了**。

### 💡 技巧一：Claude Code 的 `/compact` 命令

Claude Code 内置了一个隐藏大招：`/compact`

这个命令会**自动把当前会话的全部历史总结成一份紧凑的摘要**，然后重置对话上下文。你之前的所有讨论、代码、决策都被浓缩成一份「会议纪要」，Claude 继续帮你干活时还能记得这些关键信息，但 token 占用直接清零。

**用法超简单：**
```
/compact
```

Claude 会回复一个摘要，类似：
> 「我们讨论了用户认证模块的重构，决定用 JWT + Redis 方案，当前卡在 token 刷新逻辑的实现……」

然后你继续聊，跟什么都没发生一样流畅。

**什么时候用？**
- 感觉 Claude 开始「失忆」或重复自己
- 要聊一个全新的子任务，但不想开新窗口
- 对话已经超长篇，需要「刷新状态」

**注意**：`/compact` 只是压缩，不会丢失信息。但如果你真的想彻底清零，用 `/clear`。

### 💡 技巧二：Kimi Code 的「@多文件」快速上下文

Kimi Code 最近有个很实用的细节：在聊天框里直接 `@文件名` 或 `@文件夹名`，就能**瞬间把对应文件/目录的上下文拉进来**，不用手动复制粘贴。

比如你写：`@src/utils 这个目录下的加密函数怎么复用？`

Kimi 会直接读取整个目录结构，定位到相关文件，比你一句句描述「就是那个 third_party 下面 crypto 相关的……」高效十倍。

**进阶用法**：
- `@文件名:行号` 精准定位到具体函数
- `@README.md` 让 AI 先读项目说明，再回答你的问题
- 配合 `/explain` 命令，直接让 AI 解读你指定的代码片段
