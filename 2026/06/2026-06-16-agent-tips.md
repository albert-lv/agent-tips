# 🎯 Agent 语气密码：一句话让 AI 从"敷衍"变"认真"

> **向上 & 小惠的每日 Agent 小技巧** | 2026-06-16

---

有没有这种感觉：同一个 Agent，有时候输出质量像实习生交的作业，有时候又像 senior 在带你 review 代码——**区别往往只是一句话**。

今天分享一个几乎零成本、但能显著提升 Agent 输出质量的技巧：**给 Agent 一个"身份锚点"**。

---

## 💡 技巧：用角色设定代替泛泛的"请帮我..."

不要只说：
> "帮我写个 Python 爬虫"

试试这么说：
> "你是一位有 10 年经验的 Python 工程师，正在带新人做 code review。请写一个**生产级**的爬虫，要求：有异常处理、有日志、有 rate limiting、代码要可以直接跑。"

**区别在哪？**

| 方式 | Agent 心态 | 输出质量 |
|------|-----------|---------|
| "帮我写个..." | 随便应付一下 | 基础版，能跑就行 |
| "你是 senior engineer..." | 得拿出真本事 | 完整版，考虑边界情况 |

这个技巧在 **Claude Code**、**Cursor**、**Kimi Code**、**Windsurf Cascade** 里都一样好使。

---

## 🎮 进阶玩法：叠加"场景压力"

想要更狠的？给 Agent 加一点"场景压力"：

> "你正在面试一家顶级 AI 公司的 senior 岗位，这是面试官给你的 coding challenge。请写出**你职业生涯中最漂亮的一份答案**。"

> "这个脚本明天要上线处理 1000 万条数据，出 bug 你会被 on-call 叫醒。请确保它**健壮到可以睡觉**。"

听起来有点中二？但真的有用。LLM 对"情境"和" stakes（ stakes 高低）"非常敏感，你给的情境越具体，它越会调动"深层能力"来回应。

---

## ⚡ 快速上手模板

复制粘贴直接用：

```
你是一位[具体角色，e.g. 全栈工程师 / 算法工程师 / 技术写作者]，
正在[具体场景，e.g. 做 code review / 写技术博客 / 设计系统架构]。
请[任务]，要求：
1. [质量要求 1]
2. [质量要求 2]
3. [输出格式要求]
```

---

## 🧠 为什么这招有效？

简单说：LLM 不是搜索引擎，是**角色扮演大师**。你给的"角色剧本"越详细，它越能调用对应的知识模式和表达风格。

这就像你让朋友帮忙——你说"随便弄一下"和"这个对我很重要，拜托认真搞"，得到的结果完全不同。

---

**今日作业**：下次用 Agent 的时候，试试加一句角色设定，看看输出有没有变厚 😏
