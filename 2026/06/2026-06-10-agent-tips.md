## 技巧一：Claude Code 的终端模式——把 AI 扔进 Shell 里跑

很多人把 Claude Code 当「代码编辑器里的聊天框」用，但其实它有个**更野的打开方式**：直接在终端里跟它对话，而且它能**执行命令、读取输出、自动修复**。

### 怎么玩？

在项目根目录打开终端，直接输入：

```bash
claude
```

然后你就可以像跟朋友交代任务一样说话：

> "帮我查一下这个项目里所有用了 `TODO` 标记的地方，整理成清单给我"

Claude 会：
1. 自己执行 `grep -r "TODO" .`
2. 读取输出结果
3. 整理成 markdown 表格给你

> "测试挂了，帮我看看这个报错是什么意思"

Claude 会：
1. 运行 `pytest` 或 `npm test`
2. 读取报错日志
3. 定位到具体代码位置
4. 给出修复建议，甚至直接改好

### 什么时候比编辑器里更好用？

- **批量重构**："把 `src/` 下所有 `.js` 文件改成 `.ts`，并补上类型声明"——编辑器里你得一个个点，终端里一句话就跑了
- **环境排查**："为什么我的 Docker 容器起不来？"——Claude 能读日志、查配置、给诊断
- **CI/CD 调试**：把 GitHub Actions 的报错日志丢给它，它能在本地复现并修复

**核心优势**：Claude Code 在终端模式下有**完整的文件系统访问权限和命令执行能力**，它不只是「建议」，而是真的能「动手」。

## 技巧二：Windsurf 的 Cascade——让 AI 自己开多个线程干活

Windsurf 最近有个很离谱的功能叫 **Cascade**，它的 Agent 不是单线程跟你一问一答，而是会**自己规划任务、拆分子任务、并行执行**。

### 和普通 Agent 的区别

普通 AI 编程工具：
> 你："帮我加一个新功能" → 它改一个文件 → 你："还有这个文件也要改" → 它再改 → 反复 10 轮

Windsurf Cascade：
> 你："帮我加一个新功能" → 它自己分析出要改 5 个文件 → **同时改完** → 一次性给你看全部变更

### 实际体验

当你说：
> "给这个项目加上用户认证，包括登录、注册、JWT token 和路由保护"

Cascade 会：
1. 自动创建 `auth.py` 处理 JWT
2. 同时修改 `models.py` 加用户表
3. 修改 `routes.py` 加登录注册接口
4. 修改 `middleware.py` 加路由保护
5. 更新 `requirements.txt` 加依赖

**所有变更在一个回合里完成**，你看到的是完整的 PR 级变更，而不是"挤牙膏"式修改。

### 什么时候用？

- **功能完整开发**：从零实现一个模块，Cascade 能自己规划依赖关系
- **跨文件重构**：改一个函数签名，它自动找全项目调用点一起改
- **样板代码生成**："给我搭一个完整的 REST API 脚手架"——它直接给你生成整个目录结构

下次遇到重复性、多步骤、跨文件的任务，别自己硬啃了——交给 Agent，你负责验收就行。
