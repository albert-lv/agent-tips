## 技巧一：Claude Code 的 `/plan`——让 AI 先交方案再开工

Claude Code 有个很多人忽略的命令：`/plan`。它的作用很简单：**让 Claude 先分析、给方案、等你点头，然后再动手改代码。**

### 怎么玩？

在项目里直接输入：

```bash
/plan 帮我重构这个项目的配置管理，把环境变量和配置文件统一到一个模块里
```

Claude 会：
1. 先扫描现有代码结构
2. 给出一份**执行计划**（要改哪些文件、怎么改、顺序是什么）
3. 等你确认或调整
4. 你打 `y` 之后，它才开始执行

### 对比感受一下

**不用 `/plan`：**
> 你："重构配置管理" → 它直接改 → 你："等等，那个文件别动" → 撤销 → 重来 → 血压上升

**用了 `/plan`：**
> 你："/plan 重构配置管理" → 它给方案 → 你："第 3 步可以换个方式" → 它调整方案 → 你确认 → 它执行 → 一次到位

### 什么时候必用？

- **大范围重构**：改 5 个以上文件时，先看清全貌再动手
- **不确定需求**："帮我优化这个模块"——你自己也没想好具体怎么做，先让 Claude 出方案
- **教新人**：把 `/plan` 的结果当作一份自动生成的技术方案，直接贴在 PR 描述里

## 技巧二：Claude Code 的 `/compact`——上下文爆了别慌，压缩一下

Agent 用久了，聊天历史越来越长，Claude 开始"失忆"——前面说的要求后面忘了。这时候不是重开对话，而是 `/compact`。

### 怎么玩？

聊着聊着，感觉 Claude 开始重复问同一个问题，或者忘了你最初的指令：

```bash
/compact
```

Claude 会**自动总结**当前对话的关键信息：
- 你最初的任务是什么
- 已经做了哪些修改
- 当前遇到的问题
- 下一步要做什么

然后**清空历史**，把总结后的精简上下文保留下来，继续对话。

### 效果

- **上下文窗口回到满血状态**，Claude 又记得你最初的指令了
- **不会丢失进度**，总结里包含了所有关键信息
- **免费清内存**，不用重开对话从头讲

### 什么时候用？

- 对话超过 20 轮以上
- Claude 开始问你"你想做什么？"（明显失忆）
- 要切换子任务但不想丢失主任务上下文

## 🧠 今日思考

Agent 不是「你说一句它做一句」的工具，它其实有**策略层**和**执行层**的区别。

`/plan` 是**策略层**——让 Agent 先想清楚了再动手，你当审批者。
`/compact` 是**资源管理**——让 Agent 的「脑子」保持清醒，不被历史垃圾拖垮。

两个命令配合用：大任务先用 `/plan` 拆清楚，过程中定期 `/compact` 清内存，Agent 的稳定性直接上一个台阶。
