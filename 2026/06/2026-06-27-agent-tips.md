# 2026-06-27 Agent 工具小抄

> Claude Code 的「自动驾驶」模式：让 AI 替你盯着后台，自己修自己 🔄

## 今日技巧：/loop + Monitor，一对被低估的摸鱼神器

很多人用 Claude Code 还是一问一答模式，但其实它有两个隐藏大招，组合起来能让 agent 真的「自己跑起来」：

### 1. `/loop` —— AI 给自己写 TODO 清单

Claude Code 的 `/loop` 不是简单的重复执行，而是**让 AI 自己评估进度、自己决定下一步**。

比如你让它「把项目的所有 console.log 改成 logger.debug」，它不会一次性瞎改完，而是：
- 扫描哪些文件有 console.log
- 改一批 → 跑测试 → 看有没有报错
- 自己决定继续改还是调整策略
- 直到完成或遇到无法处理的 blocker 才停下来

**关键**：它会在每次循环后给你一个状态摘要，你可以随时 `Ctrl+C` 接管，不会失控。

### 2. `Monitor` —— 后台进程「直播弹幕」

Claude Code 2.1 新增的 Monitor 工具，可以在**后台实时流式输出进程日志**。

实际场景：
```bash
# 你让 Claude 跑一个耗时 10 分钟的测试套件
claude > 运行测试套件，如果有失败就修

# 它会启动测试，同时开 Monitor 盯着输出
# 你可以切去干别的，它在后台自己看着
# 测试挂了？它会自动读取错误日志，定位文件，改代码，重跑
```

**Monitor 的妙处**：不像以前只能等进程跑完再看输出，现在是**边看边处理**，有问题秒修，不用你反复 `claude > 测试又挂了帮我看看`。

### 3. 组合技：真正的「晚上提需求，早上收结果」

最猛的用法是：**`/loop` + `Monitor` + Git Worktree**（Claude Code 支持多 worktree 并行）。

- 在一个 worktree 让 Claude 跑重构（用 /loop 自己迭代）
- 另一个 worktree 正常写你的需求
- Claude 会自己提交中间 commit，有问题自己 rollback

第二天早上回来：分支上有 5 个干净的 commit，每个都有描述，测试全绿。你只需要 review 和 merge。

## 一句话总结

Claude Code 的 `/loop` 不是让你偷懒，而是让你从「盯着 AI 干活」变成「定好目标，回来验收」。

 Monitor 让你不用担心 AI 在后台瞎搞——它在看着呢。
