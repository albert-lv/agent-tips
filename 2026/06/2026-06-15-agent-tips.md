# 用 `@` 符号精准定位：让 Agent 别在错误文件里瞎改

**日期：2026-06-15 | 第 6 期**

---

## 1. `@` 符号：Agent 的"导航仪" ⭐

你有没有遇到过这种情况：你跟 Claude Code 说"改一下这个函数"，结果它跑去改了一个同名但完全无关的文件，还自信满满地告诉你"搞定了"。

这就是 Agent 的 **"幻觉式文件定位"**——它确实"知道"项目结构，但有时候它的"知道"和实际的"文件路径"对不上号。

**解决方案：在提示词里用 `@` 明确指定文件。**

比如：

```
@go/pkg/dashboard/server.go 里的 handleMetrics 函数，
改成返回 JSON 格式而不是纯文本。
```

Claude Code、Kimi Code、Windsurf Cascade 都支持这种 `@文件名` 的写法。它相当于给 Agent 装了一个导航仪：

- **不带 `@`**：Agent 自由发挥，凭"记忆"找文件（可能走错）
- **带 `@`**：Agent 直接锁定到该文件，上下文更精准，改动更靠谱

### 实战技巧

1. **批量修改时列清单**：`@file1 @file2 @file3，这三个文件统一把 fmt.Println 换成 log.Info`
2. **跨文件引用时**：`@server.go 里的 handler 和 @client.go 里的 caller，接口对齐一下`
3. **对比查看时**：`@v1.go 和 @v2.go 有什么区别？`

## 2. 配置文件"埋记忆"：让 Agent 记住你的规矩

如果你每次开新会话都要重复"我们用 Go 1.22""别用全局变量""错误处理用我们的 wrap 包"——那你需要给 Agent 写一份 **"入职手册"**。

每个 Agent 工具都有类似的"长期记忆"文件：

| 工具 | 配置文件名 |
|------|-----------|
| Claude Code | `.claude.md`（放在项目根目录） |
| Cursor | `.cursorrules` |
| Windsurf | `.windsurfrules` |
| Kimi Code | 一般支持 `.cursorrules` 或类似机制 |

### 在 `.cursorrules` 里写点什么？

```markdown
# 项目规范
- 使用 Go 1.22，所有代码必须过 go vet
- 错误处理统一用 github.com/albert-lv/pkg/errors 的 Wrap 方法
- 不写全局变量，依赖通过构造函数注入
- 测试覆盖率低于 80% 的 PR 不允许合并

# 代码风格
- 函数长度不超过 50 行，超过就拆分
- 变量命名用 camelCase，不要缩写
- 注释用英文，面向国际社区

# 常见陷阱
- 不要直接操作数据库，必须通过 repository 层
- HTTP handler 里不要写业务逻辑，只负责解析和调用 service
```

Agent 每次读取项目时都会先看这个文件，相当于 **"你提前给它培训好了"**。新会话里它不会问"咱们用啥 Go 版本"，不会乱改风格，不会踩你早提醒过的坑。

---

## 一句话总结

> `@` 管当下——让 Agent 改对文件；配置文件管长远——让 Agent 记住你的规矩。两个一起用，Agent 从"看起来会写代码的实习生"变成"带了三个月已经熟门熟路的搭档"。
