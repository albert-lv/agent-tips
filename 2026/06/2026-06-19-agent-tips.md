# 🚀 今日 Agent 技巧 — 2026-06-19

## 技巧：让 Agent 帮你「读」代码，而不是「写」代码

很多人把 Claude Code / Kimi Code / Cursor 当自动代码生成器用，但其实它们的**最强用法是：当你的眼睛和大脑的外挂。**

### 具体怎么做？

**1. 进一个陌生的代码库，先问它：**

> "这个仓库的整体架构是什么？核心模块之间的关系画个图给我。"

Claude Code 会帮你梳理出调用链、数据流、关键抽象。不用你自己翻三小时目录结构。

**2. 遇到一段看不懂的祖传代码：**

> "这段函数在干什么？输入输出是什么？有什么副作用？"

比你自己打断点、加 console.log 快 10 倍。尤其适用于那种「没有注释、变量名是 a/b/c、但业务逻辑极其重要」的代码。

**3. 看 PR 之前，让 Agent 先过一遍：**

> "总结一下这个 PR 改了什么，有没有潜在风险点？"

Claude Code 的 `/review` 命令就是干这个的。你可以直接问它："这个改动会破坏向后兼容吗？" —— 比人眼更不容易漏边界情况。

### 为什么这比「让 Agent 写代码」更值？

因为**读代码的时间占你工作的 70%**，写只占 30%。Agent 帮你读得快、读得准，省下来的时间才是大头。

而且读代码的时候你人在掌控，Agent 只是辅助解释；写代码的时候它可能瞎写你还得擦屁股。

## 💡 一句话总结

> 把 Agent 当成「带超链接的代码向导」，而不是「自动打字机」。

今天试试进一个你不太熟的仓库，先问 Agent "这个项目的架构是什么？" —— 你会惊讶它说得有多清楚。
