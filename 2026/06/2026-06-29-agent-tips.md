# 每日 Agent 技巧 — 2026-06-29

> 今天聊一个 Claude Code 的隐藏操作 + Cursor 的"反向提问"技巧。

---

## 🎯 技巧一：Claude Code 的 `/load` 指令 — 让 Agent 秒懂你的代码库

很多人用 Claude Code 是直接进入就开始问问题，结果 Agent 在那里一顿猜你的项目结构。

其实 Claude Code 有个 `/load` 指令，可以直接加载一个文件或目录的上下文：

```
/load src/
```

或者加载特定文件：

```
/load README.md package.json
```

**重点来了**：`/load` 不只是把文件内容塞进去，它会让 Claude Code 主动**分析这些文件的关联关系**。比如你把 `package.json` 和 `src/` 一起加载，它会自动识别依赖项和实际代码的映射。

**实战场景**：
- 刚进一个新项目，先 `/load README.md` 再开始提问
- 要改某个模块，先 `/load src/that-module/` 让 Agent 聚焦
- 排查 bug 时，`/load` 相关测试文件，Agent 会结合测试逻辑分析

💡 **小提示**：`/load` 后可以直接跟自然语言指令，比如 `/load src/ "这个模块的入口在哪里？"`

## 🎯 技巧二：Cursor 的 "反向提问" — 让 AI 帮你发现你没问的问题

Cursor 有个被低估的功能：**让 Agent 主动向你提问**。

在 Cursor Composer 或 Chat 里，你可以这样说：

> "在给我方案之前，先问我 3 个关于这个需求的关键问题"

或者更具体：

> "你觉得实现这个功能还需要确认什么？列出来问我"

**为什么这招好用？**

因为 AI agent 工具最大的问题不是"答不对"，而是"问不准"。很多时候你自己都没想清楚需求边界，Agent 就开始写代码了，结果改来改去。

让 Agent 先反向提问，相当于**免费的需求梳理**。它会把那些"你忘了说但很重要"的点挖出来：
- "这个 API 的鉴权方式是什么？"
- "用户输入需要验证哪些边界情况？"
- "错误处理是要重试还是直接报错？"

**进阶玩法**：
把 Agent 的回答收集起来，形成一份"需求确认清单"，下次直接贴给 Agent，减少来回沟通。

---

## 🎬 一句话总结

> `/load` 解决"Agent 不知道我在干什么"的问题，反向提问解决"我也不知道我在干什么"的问题。两个一起用，效率翻倍。
