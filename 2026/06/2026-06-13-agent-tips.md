## 技巧一：让 Agent 读「天书」—— legacy 代码翻译器

有没有遇到过这种场面：接手一个三年前的项目，没有文档，注释少得可怜，变量名像 `a1`、`tmp2`、`handler_v3_final`，逻辑绕得像毛线团？

别硬啃，直接让 Agent 当翻译。

### 在 Claude Code / Kimi Code 里这样问：

> "帮我理解这个函数的真正意图，不要翻译代码，告诉我它在业务层面想做什么"

**关键区别：** 如果你说"解释一下这段代码"，AI 会逐行给你翻译 `if` 和 `for`，输出一大段废话。但你说"在业务层面想做什么"，它会跳过语法细节，直接告诉你：「这个函数是在做订单状态机的幂等校验，防止重复支付回调」。

### 进阶用法：让 AI 画「调用地图」

> "从入口函数开始，追踪这个请求完整的数据流，输出一个带关键节点说明的文本流程图"

Agent 会：
1. 找到入口函数
2. 顺着调用链一路追踪
3. 标记出每个关键转换点（比如"这里从 JSON 转成领域对象"、"这里做了权限校验"）
4. 用 ASCII 或 Mermaid 格式给你一张图

**效果：** 你 5 分钟获得一张别人啃了三天才画出来的流程图。

## 技巧二：跨语言「转译」——让 Agent 当技术翻译官

这个场景更实用：你懂 Python，但项目里有一段 Go 的并发逻辑；你懂 JavaScript，但后端同事扔给你一段 Rust 的匹配逻辑。手动查文档太慢，让 Agent 直接「转译」成你熟悉的语言风格。

### 实操示例：Go channel → Python asyncio

在 Cursor 或 Claude Code 里：

> "这段 Go 代码用了 channel 和 goroutine，请把它的核心并发模式翻译成 Python asyncio 的等价实现，并解释关键差异点"

**为什么比查文档快？**
- 文档会告诉你 `asyncio.Queue` 的 API 参数
- Agent 会告诉你："Go 的 `select` 对应 Python 的 `asyncio.wait()`，但 Go 的 channel 是双向通信，Python 的 Queue 是单向的，这里需要额外处理完成信号"

### 还能这么用：

- "把这段 bash 脚本翻译成 Python，并告诉我哪些边缘情况原脚本没处理"
- "这段 SQL 用了窗口函数，用 Pandas 怎么实现同样的逻辑？"
- "这个 C 的位运算操作，在 Python 里怎么写等价但可读性更好的版本？"

## 🧠 今日思考

Agent 工具最被低估的能力不是「写代码」，而是**「快速建立理解」**。人类读陌生代码库的平均时间是 2-3 天，Agent 可以在 10 分钟内给你一个可导航的地图。

下次遇到：
- 没有文档的老项目
- 看不懂的跨语言逻辑
- 需要快速评估一个开源库是否适合自己用

别自己硬啃，先丢给 Agent 做「第一遍翻译」，你再基于它给的地图做深入判断。省下来的时间，用来写新代码不香吗。
