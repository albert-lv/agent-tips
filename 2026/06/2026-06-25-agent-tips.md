# 🎯 每日 Agent 技巧 · 2026-06-25

## 技巧一：给 Agent 写一份「入职手册」

如果你每次开新项目都要跟 AI 解释一遍"我们用 Python、缩进 4 空格、不要给函数加类型注解"……那你需要一份 **规则文件**。

**Cursor** → 在项目根目录放 `.cursorrules`  
**Claude Code** → 放 `CLAUDE.md`  
**Windsurf** → 放 `.windsurfrules`

这些文件会在每次对话时自动注入上下文，相当于给 Agent 的**入职培训文档**。

### 写啥内容？参考这个模板：

```markdown
# 项目规范

## 技术栈
- Python 3.11 + FastAPI
- 数据库：PostgreSQL，ORM 用 SQLAlchemy

## 代码风格
- 缩进：4 空格
- 行宽：100 字符
- 函数必须写 docstring

## 禁止事项
- 不要用 * 导入
- 不要提交 print 调试语句
- 不要改 requirements.txt 除非我同意

## 目录结构
- /app        → 业务代码
- /tests      → 测试文件（pytest）
- /scripts    → 工具脚本
```

**效果**：Agent 不再每次问你"要不要加类型注解"，也不会擅自改你的依赖——因为入职手册里写清楚了。

## 技巧二：Claude Code 的 `/clear` 不是清屏，是「换脑子」

Claude Code 有个隐藏大招：`/clear`

你以为它是清屏？不，它是**重置上下文**。执行后，Claude 会忘记这轮对话里你提到的所有文件和代码，相当于换了一个"新鲜脑子"来帮你。

**什么时候用？**
- 聊了 20 轮后，Agent 开始"犯迷糊"，改 A 文件的时候把 B 文件搞乱了
- 想从一个完全不同的角度重新审视问题
- 上下文太长了，响应变慢

**配合技巧**：`/clear` 之后立刻说 `/init` 或打开新文件，Agent 会基于当前文件状态重新加载上下文，避免带着上一轮的思维定式。

> 💡 **Pro tip**: 如果你发现 Agent 开始"执着"于某个错误方案，怎么纠正都不听——`/clear` 比跟它吵 10 轮有用得多。这叫"上下文中毒"，换脑子是最快的解药。

---

**今日一句话总结**：规则文件省口水，`/clear` 救脑子。Agent 工具用得好，不是你会下什么指令，是你懂怎么管理它的"记忆"。
