# 🎯 每日 Agent 技巧 · 2026-07-05

> **今日主题：让 AI 替你「看」代码 —— 视觉模型 + Agent 的隐藏玩法**
>
> 适合：Claude Code、Kimi Code、Cursor、Windsurf 用户

---

## 技巧一：截图即 PR Review

别再用文字描述 UI bug 了。直接把截图丢给 Agent，说：

> "这张截图里，按钮和文字没对齐，帮我找出对应的 CSS/组件文件并修一下。"

**为什么好用：**
- Claude 3.5/4、Kimi K2、GPT-4o 都有视觉能力，能「看懂」界面
- 省去你写一大段 "在第 X 行附近，有个 Y 元素，它的 Z 属性..."
- 对前端调整、UI 细节、配色问题尤其有效

**进阶玩法：**
把设计稿（Figma 导出图）和当前页面截图一起发，让 Agent 做 "找不同"，自动对齐设计。

---

## 技巧二：用 `\describe` 或 `@` 让 Agent 读图（Kimi Code / Claude Code）

部分 Agent 工具支持特殊的图片引用语法：

- **Claude Code**: 直接粘贴图片，或拖进终端，Claude 会自动读取
- **Kimi Code**: 支持在对话中上传图片，Kimi 会分析图中的代码、报错、UI 等
- **Cursor / Windsurf**: 在 chat 里粘贴图片，模型会自动处理（需要视觉模型支持）

**实战场景：**
1. 报错截图 → "这个报错什么意思，怎么修？"
2. 日志截图 → "帮我从这段日志里找关键错误"
3. 架构图 → "根据这张图，帮我实现左侧的模块"
