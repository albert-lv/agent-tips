# 让 Agent 自己「多想一层」

> 2026-07-13 | 每日 Agent Tips

你有没有发现，有时候让 agent 写代码，它给出的方案总是……**刚好能跑，但细看有点糙**？

就像那种"应付作业"式的回答——语法没错，逻辑通顺，但缺了点工程上的考量。

不是它不行，是你没让它**多想一层**。

---

## 技巧：主动开启「追问模式」

大部分 agent 工具（Claude Code、Kimi Code、Cursor、Windsurf）默认是"快速响应"模式：你问啥它答啥，不多不少。

但你可以用一个简单的 prompt 切换它的思考深度：

### 追问三连（收藏这个模板）

```
在给出方案之前，请先：
1. 列出这个方案的潜在风险和 edge cases
2. 考虑至少一个替代方案，并简要对比
3. 如果我的需求有隐含前提（比如性能敏感、需要兼容旧版本），请指出来
```

把这个加在你的需求前面，agent 的输出质量会明显上一个台阶。

### 进阶版：针对代码评审

```
审这段代码时，不要只找 bug。请从这几个维度检查：
- 是否有隐藏的并发问题？
- 错误处理是否完整？（别只 catch 不处理）
- 命名是否真的表意？（有没有 foo/bar 这种临时感）
- 如果三个月后的自己来看，哪里会看不懂？
```

---

## 为什么这招好用？

| 场景 | 普通模式 | 追问模式 |
|------|---------|---------|
| 写 API 接口 | 返回 200/404 就完事 | 会提醒你考虑 rate limit、idempotency |
| 重构代码 | 直接搬过来 | 会问你"这个变量名是不是该改？" |
| 排查 bug | 给一个可能原因 | 列出排查路径，甚至让你加日志验证 |

**核心区别**：普通模式是"问答"，追问模式是"对话"。

---

## 一句话总结

Agent 不是不会深想，是默认在等你的许可。**你问得浅，它就答得浅；你要求它多想一层，它真的能多想一层。**

下次觉得 agent 在"敷衍"，试试给它加个「追问三连」👆
