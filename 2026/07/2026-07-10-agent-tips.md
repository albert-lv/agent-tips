# 🛠️ Agent Tips 2026-07-10

## 今日技巧：Agent 的「上下文压缩」艺术

### 1. Claude Code 的 `/compact` 指令 —— 超长会话的救星

你有没有遇到过这种情况：跟 Claude Code 聊了一个小时，项目背景、需求变更、各种试错全堆在上下文里，然后它开始「失忆」或者「幻觉」？

Claude Code 其实内置了一个 `/compact` 命令，专门解决这种「上下文膨胀」问题：

- **作用**：把当前会话的所有历史自动压缩成一份简洁的摘要，同时保留关键代码上下文
- **时机**：当你感觉它开始重复提问、理解偏差、或者项目背景太复杂时
- **效果**：相当于给对话按了「刷新」按钮，但重要信息不会丢

> 用法：直接输入 `/compact`，它会自动生成摘要并提示你是否确认压缩。

### 2. Kimi Code 的 `@file` 技巧 —— 精准投喂上下文

跟 Agent 聊天最怕「给太少它不懂，给太多它迷糊」。Kimi Code 的 `@file` 引用支持**行号范围**和**符号跳转**，这个很多人没用到：

- `@file:src/main.rs:10-30` → 只投喂 10-30 行，精准打击
- `@file:src/main.rs#MyStruct` → 直接跳转到 `MyStruct` 定义，不用翻页
- 多文件时：`@file:A.rs @file:B.rs` 组合引用，Agent 会自动建立关联

这个技巧在代码审查和重构时特别有用——你不需要把整个文件丢过去，只需要给它「看到问题」的最小上下文。

---

💡 **一句话总结**：Agent 不是越给越多就越好，学会「压缩」和「精准投喂」，上下文利用率能翻倍。
