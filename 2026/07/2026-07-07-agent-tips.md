# 🤖 Agent 技巧 · 2026-07-07

## 今日主题：别再用 "请帮我看看这段代码" 了——@ 一下，Agent 秒懂你

大家都知道 Claude Code / Cursor / Kimi Code 能读文件，但很多人还在用"请帮我看看 `src/utils.py`"这种原始人沟通法。

其实有更香的方式 👇

---

### 💡 技巧：用 `@` 精确投喂上下文

**Claude Code / Cursor / Kimi Code 都支持文件/符号级别的 `@` 引用。**

| 输入方式 | 效果 |
|---------|------|
| `@src/utils.py` | Agent 直接读取整个文件 |
| `@getUserData` | 直接定位到函数/类的定义，不用翻整个文件 |
| `@#L45-L60` | 只引用某几行，避免上下文爆炸 |

**为什么这很香：**

普通提问：
> "帮我看看 `auth.js` 里的登录逻辑有没有 bug"

Agent 内心 OS：*auth.js 300 行，你让我猜是哪段？*

进阶提问：
> "`@validateToken` 这个函数在 `@auth.js` 里，如果 token 过期了会抛什么错误？能不能优雅处理？"

Agent 内心 OS：*懂了，直接锁定目标，开干。*

---

### 🎯 Bonus：组合引用 = 降维打击

试试这个：

> "对比 `@UserService.login` 和 `@OAuthService.authenticate` 的错误处理逻辑，看看能不能统一成一个 `@AuthError` 类？"

Agent 会：
1. 读取两个服务的代码
2. 对比错误处理差异
3. 提出重构方案 + 代码

**你花 10 秒写 prompt，Agent 花 30 秒出方案。** 这波叫"用上下文精度换输出质量"。

---

### ⚠️ 注意

- `@` 不是万能的——如果文件太大（比如几千行的生成代码），还是切几行引用更靠谱
- 不同工具语法略有差异，Claude Code 用 `@file`，Cursor 用 `@` 或 `Ctrl+Enter` 选上下文，Kimi Code 也支持类似能力

---

**一句话总结：** 精确引用 > 模糊描述。Agent 不是读心术，但你用 `@` 喂得越准，它回得越狠。
