## 技巧一：Cursor 的 `Ctrl+K` 不只是聊天

很多人把 `Ctrl+K` 当聊天框用，其实它是个**上下文感知的快速编辑神器**。

**正确打开方式：**
1. 选中一段代码，按 `Ctrl+K`
2. 不用复制粘贴，直接说："加个异常处理"、"改成异步的"、"这坨太丑了重构一下"
3. Cursor 会**只修改你选中的部分**，而不是全场乱改

**高阶玩法：**
- 选中函数名 + 注释 → 让 AI 帮你补全实现
- 选中测试文件 → "给这个补两个边界 case"
- 选中错误日志 → "这报错什么意思，怎么修"

> 💡 **为什么香**：它比全局 chat 更精准，比手动改更快，属于那种"用了就回不去"的功能。

## 技巧二：Claude Code 的终端集成，把 AI 变成你的 shell 搭档

Claude Code 不只是写代码，它能**在你的终端里真的干活**。

**几个实用场景：**

| 你说 | 它干 |
|------|------|
| "run the tests" | 执行测试，分析失败原因 |
| "check git status" | 看改动，还能帮你写 commit message |
| "find where X is defined" | 比 grep 聪明，能跨文件找逻辑关联 |
| "deploy to staging" | 执行部署命令，监控输出 |

**关键认知：** Claude Code 的 `\>` 模式不是"问 AI 问题"，而是"指挥一个懂代码的实习生"。你可以让它**连续执行多步操作**，比如：

```
> 先跑测试，如果有失败的就修，修完提交
```

它会真的按步骤执行，中途问你确认。

> ⚠️ **安全提醒**：它确实能执行命令，所以**重要操作前看清楚它要执行什么**，别让它 `rm -rf /` 了（虽然 Claude 一般不会这么干，但养成检查的习惯）。
