# Agent 的「外接硬盘」大法

> 2026-07-12 | 每日 Agent Tips

你有没有遇到过这种情况：让 Claude Code 改一个功能，它吭哧吭哧改了 20 个文件，改到一半突然问你「我们刚才要干嘛来着？」

不是它菜，是**上下文窗口满了，早期记忆被挤掉了**。

就像你脑内开了 20 个 tab，最开始的那个早就不知道跑哪去了。

## 技巧：给 Agent 配一个「外接硬盘」

核心思路很简单：**不要让 agent 把所有事情都记在脑子里，给它一个文件当备忘录。**

### 具体做法

**1. 开干之前，先建一个 `TODO.md`**

```bash
# 在 Claude Code / Kimi Code 里直接说：
"先创建一个 TODO.md，把我们今天要做的几件事列出来，每完成一件就打勾"
```

**2. 让 agent 自己维护这个文件**

好的 agent 工具（Claude Code、Kimi Code、Cursor Composer）都支持文件读写。告诉它：

> "每完成一个步骤，更新 TODO.md 标记进度。如果我发现你跑偏了，我会指出来，你要把纠正后的方向也写进 TODO.md。"

**3. 复杂任务再加一个 `THOUGHTS.md`**

如果任务特别复杂（比如重构一个模块、排查一个 bug），让 agent 把**关键决策和中间结论**也落盘：

```markdown
## 排查记录
- 14:32 排除了数据库问题，慢在 API 层
- 14:45 发现是 N+1 查询，准备上 prefetch
- 15:10 prefetch 生效，但内存涨了，需要权衡
```

这样即使上下文被压缩，agent 扫一眼这两个文件就能秒回状态。

## 为什么这招好用？

| 场景 | 不用外接硬盘 | 用了外接硬盘 |
|------|-----------|-----------|
| 跑了 30 分钟突然中断 | "从头再说一遍？" | 看 TODO.md，接着干 |
| 你中途去开了个会 | agent 失忆 | 文件还在，状态永存 |
| 多轮纠偏后方向混乱 | 越改越歪 | THOUGHTS.md 里有决策日志 |

## 一句话总结

**Agent 的脑子会满，但文件系统不会。** 善用「外接硬盘」，让你的 agent 从金鱼变成 elephant 🐘
