# 每日 Agent 技巧 ｜ 2026-07-08

## 🎯 今日主题：让 Agent "记住" 你的习惯

大家有没有发现，每次跟 AI Agent 开始新会话，都要重新交代一遍自己的偏好？比如"我喜欢用中文变量名""请把代码写紧凑一点""别给我写注释"……

今天分享两个让 Agent 真正"懂你"的技巧 👇

---

## 1️⃣ Claude Code：`.claude.md` 项目级系统提示

Claude Code 最近支持了一个很实用的功能——在项目根目录放 `.claude.md` 文件，里面写的所有内容都会自动注入到每次对话的 system prompt 里。

**用法超简单：**

```markdown
# .claude.md

- 代码风格：优先简洁，注释只解释"为什么"不解释"是什么"
- 命名习惯：中文项目用英文变量，保持语义清晰
- 技术栈：Go 用标准库优先，Python 用 type hints
- 特殊要求：改代码前先跑测试，别破坏现有用例
```

然后每次启动 `claude` 命令，它就知道你的规矩了。不用每次重复，也不用怕它忘。

💡 **进阶玩法**：不同项目可以有不同的 `.claude.md`，工作项目一个风格，个人项目另一个风格，互不干扰。

---

## 2️⃣ Cursor Rules：用 `.cursorrules` 做团队规范

Cursor 的 `.cursorrules` 文件跟上面的思路类似，但有个更狠的用法——**可以写规则让 AI 主动检查代码质量**。

比如这样写：

```markdown
# .cursorrules

## 代码提交前检查清单
- [ ] 是否有未处理的 error？不要吞掉错误
- [ ] 新增函数是否有对应测试？
- [ ] 是否引入了新的依赖？必要性是否成立？
- [ ] 命名是否符合项目惯例？

## 对话风格
- 先理解需求，再写代码
- 遇到不确定的地方直接问，别猜
- 改完代码后简要说明改动点
```

Cursor 读到这些规则后，会在生成代码时自动对齐你的标准，甚至会在你觉得"差不多了"的时候提醒你："等等，你好像没写测试？"

---

## 🤝 为什么这招管用

本质上是把"每次都要说的话"沉淀成"默认就有的上下文"。Agent 的上下文窗口再长，也架不住你每次从头交代。把偏好写成文件，相当于给 Agent 装了一个"出厂配置"。

而且 `.claude.md` 和 `.cursorrules` 都可以进版本控制，团队成员拉下代码就自动继承统一规范，比写文档、开会说一百遍都管用。
