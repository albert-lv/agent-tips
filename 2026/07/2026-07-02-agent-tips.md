# 2026-07-02 Agent 早茶 🍵

---

## 技巧一：Kimi Code 的 `continue` / `@` 续接

你有没有遇到过这种情况：让 Kimi Code 改一段代码，改到一半它停了，说"由于长度限制，我分两部分输出"——然后第二部分你死活等不来？

**解法：** 直接回车发个空格，或者打字 `@` 再回车，它就知道你要"继续刚才的"。不需要重复描述需求，上下文是连着的。

同理，Claude Code 里也有类似机制：如果输出被截断，直接说 "continue" 或 "go on"，它会无缝接上。Cursor 和 Windsurf 也一样，只是触发词可能不同（Cursor 常用 `continue`，Windsurf 用 `@` 或空回复）。

**💡 进阶玩法：** 如果你发现它"继续"的时候偏离了原来的方向，可以在空回复前先补一句"按刚才的思路继续，不要改方案"——这个小锚定能大幅降低走偏概率。

---

## 技巧二：Agent 的"上下文窗口焦虑"——用文件做外脑

所有 Agent（Claude Code、Kimi Code、Cursor、Windsurf）都有一个共同瓶颈：**上下文不是无限的**。聊久了，它开始"失忆"，忘记你 20 分钟前定的规则。

**解法：把规则写进文件，让 Agent 自己读。**

- Claude Code：`.claude/CLAUDE.md` 或项目根目录的 `CLAUDE.md`
- Kimi Code：`.kimi/KIMI.md`
- Cursor：`.cursorrules`
- Windsurf：`.windsurfrules`

在这些文件里写上你的项目规范、代码风格、常用命令——Agent 每次启动都会读到，比你反复口头叮嘱稳定 10 倍。

**🎯 一句话总结：** 口头约定容易忘，文件约定不会丢。
