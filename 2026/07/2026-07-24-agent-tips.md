# 🎯 Agent 的「注意力管理」——别让 AI 在信息海洋里溺水

> 2026-07-24 | daily agent tips

你有没有发现——

跟 Agent 聊得越久，它越像那个**边回微信边听你说话**的朋友：

> "嗯嗯……等等你刚说啥？这个文件是干嘛的来着？"

不是它不行，是上下文太长，AI 也犯迷糊。今天聊两个「注意力聚焦」技巧 👇

---

## 技巧一：Claude Code 的 `/compact` —— 手动给大脑腾地方 🧹

**痛点：** 会话开了两小时，改了十几个文件，AI 开始「幻觉」——提到不存在的代码、记错你半小时前说的需求。

**解法：** 输 `/compact`。

Claude Code 会把之前的对话历史压缩成一份摘要，扔掉废话，保留关键决策和当前状态。相当于给 AI 做了一次「脑容量整理」。

**什么时候用：**
- 改了 N 个文件后，要进入新任务模块
- AI 开始重复问已经确认过的事
- 你想换个话题，但又不想开新会话（因为当前上下文里有重要信息）

**进阶combo：**
先 `/cost` 看看烧了多少 token，再 `/compact` 清一波，最后继续干活。省钱+清醒，双赢。

> 💡 小总结：`/compact` 是 Claude Code 的「深呼吸」——不是重启，是整理后继续。

---

## 技巧二：Cursor 的「精准投喂」——只给 AI 看该看的 📌

**痛点：** 你问 AI 一个关于 `utils.ts` 的问题，它却开始分析 `package.json`、`README.md`、甚至三年前写的测试文件……上下文乱成一锅粥。

**解法：** 别让它自己猜，你主动「圈定范围」。

**三种投喂方式：**

**1. 选中代码按 `Ctrl+L`（或 `Cmd+L`）**
只把选中的几行代码丢进 Chat，其他文件一概不带。适合"这段逻辑怎么优化"的精准提问。

**2. 用 `@` 点名具体文件**
```
你：@utils.ts 里的 parseDate 函数，帮我加上时区支持
```
Cursor 只会读 `utils.ts`，不会四处乱逛。

**3. Agent 模式下的「Exclude」**
在 Cursor Agent 的上下文列表里，右键点击不相关的文件 → "Exclude from context"。AI 就会假装这些文件不存在。

> 💡 小总结：Agent 不是越博学越好，有时候「故意让它无知」才是高效。只给必要的上下文，答案更准、token 更省。

## 今日冷知识 🧊

Kimi Code 其实也有类似的聚焦能力——在侧边栏聊天时，点击编辑器右上角的 **「引用当前文件」** 按钮（一个小回形针 📎 图标），可以锁定 AI 只关注当前打开的文件，而不是整个工作区。

适合那种"我就问这一页的事，别扯别的"的场景。

## 一句话总结

> 好 Agent 不是记得越多越好，而是**该记得记得住，该忘掉的别占内存**。

就像你不可能同时听五个人说话还都记住，AI 也一样。帮它聚焦，它才能真帮到你。
