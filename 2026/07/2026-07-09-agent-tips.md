# 🛠️ Agent 工具每日技巧 — 2026-07-09

> 今天聊两个最近实测很香的玩法：Claude Code 的「Plan Mode」和 Cursor 的「Background Agent」。
> 不是那种"你听说了吗"的八卦，是真的能让你少踩坑、少返工的具体操作。

---

## 技巧一：Claude Code 的 Plan Mode —— 先想后干，少翻车

### 是什么

Claude Code 的 **Plan Mode**（计划模式）= 让 AI 先给你一份完整的执行方案，等你点头确认后，它才真正动手改代码。

### 怎么用

```bash
# 在 Claude Code 里按 Shift+Tab 两次，进入 Plan Mode
# 然后下任务，比如：
claude "帮我重构这个模块，把所有的同步调用改成异步"
```

AI 会：
1. **分析** 当前代码结构和依赖关系
2. **列出** 需要改动的文件清单和具体修改点
3. **等你确认** — 你可以删减、调整优先级
4. **执行** — 确认后才动手，避免"改到一半发现方向错了"的尴尬

### 什么时候用

| 场景 | 为什么用 Plan Mode |
|------|-------------------|
| 跨多个文件的重构 | 先看清全貌，避免改完 A 发现 B 又崩了 |
| 涉及 API 变更 | 确认接口兼容性后再动手 |
| 你不确定 AI 理解对了需求 | 先审方案，再审代码 |

### 一句话总结

> "让 Claude 先画图，再动工。不是不信任，是尊重自己的血压。"

## 技巧二：Cursor 的 Background Agent —— 让它后台跑着，你干别的

### 是什么

Cursor 的 **Background Agent**（后台 Agent）= 你派一个 Agent 去执行耗时任务（比如跑测试、生成文档、分析代码库），它自己在后台干活，不卡你的编辑器。

### 怎么用

```bash
# 在 Cursor 的 Composer 里，用自然语言描述任务：
"分析这个项目的所有 API 路由，生成一份接口文档，存到 docs/api.md"

# 然后选择 "Run in Background"
```

Cursor 会：
- 在后台独立进程里运行 Agent
- 你可以继续写代码、切文件、甚至关掉 Composer 面板
- 完成后通知你，直接看结果

### 什么时候用

| 场景 | 为什么用 Background Agent |
|------|--------------------------|
| 代码库分析（比如"找出所有 TODO"） | 分析几百个文件时，不阻塞你干活 |
| 批量重构（比如"统一错误处理风格"） | 让它慢慢跑，你去喝咖啡 |
| 生成文档 / 测试用例 | 耗时但不急，后台搞定即可 |

### 一句话总结

> "Cursor 的 Background Agent 就像雇了个实习生：你派活，它默默干，干完敲门汇报。你全程不用盯着。"

## 组合技：Plan + Background = 王炸

一个进阶用法：

1. 用 **Claude Code Plan Mode** 设计复杂重构的方案
2. 把方案里的子任务拆给 **Cursor Background Agent** 并行执行
3. 你在中间 review 结果、调方向

相当于：**Claude 当架构师画蓝图，Cursor 当施工队同时开工。**
