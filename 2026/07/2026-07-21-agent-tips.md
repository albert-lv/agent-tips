## 技巧一：Claude Code 的 `/cost` —— 别再盲猜花了多少钱 💸

**痛点：** 用 Claude Code 处理大项目，聊了一上午，心里没底："这会儿烧了多少钱？"

**解法：** 直接输入 `/cost`。

它会给你一个清晰的账单：
- 本次会话用了多少 input/output token
- 换算成美元大概多少
- 累计今天的总花费

**真实场景：**
> 你让 Claude Code 重构一个 30 文件的模块，它吭哧吭哧改了半天。你有点慌，输个 `/cost` 一看——才 $0.12。好，继续。😌

**进阶玩法：**
- 结合 `/compact` 手动压缩上下文，能显著降低 token 消耗
- 对重复性任务（比如批量改命名），先让 AI 写脚本，比直接改省钱 10 倍

> 💡 小总结：大钱花在刀刃上，`/cost` 就是那个让你心里有数的刀刃。

## 技巧二：Cursor 的 `@Codebase` —— 全库搜索，指哪打哪 🎯

**痛点：** 想改一个函数，但不知道它在哪里被调用了，手动搜又怕漏。

**解法：** 在 Cursor Agent 里输入 `@Codebase`，然后描述你要找的东西。

Cursor 会：
1. 自动分析整个代码库的语义
2. 找出最相关的文件和函数
3. 直接把上下文喂给 AI

**示例对话：**

```
你：@Codebase 帮我找到所有调用 sendEmail 的地方，我想改成异步的

Cursor：找到 5 处调用，分布在 3 个文件里。建议先把接口改成 async，然后逐个处理……
```

**和纯文本搜索的区别：**
- `Ctrl+Shift+F` 只能匹配字符串
- `@Codebase` 理解语义，能找出 "sendEmail" 的别名、封装层、间接调用

> 💡 小总结：`@Codebase` 是 Cursor 的「全图视野」，适合接手陌生项目或做大范围重构。

## 今日冷知识 🧊

Windsurf 的 Cascade 其实支持 **自定义系统提示词**——在设置里找到 "Custom Instructions"，你可以给 Cascade 定个专属人设，比如：

> "你是一个谨慎的代码审查员，每次修改前必须先解释风险点。"

这样它就不会无脑冲了，适合那种"宁可慢点也别搞崩"的关键代码。
