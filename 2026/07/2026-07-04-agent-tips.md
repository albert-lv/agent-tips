## 技巧一：Claude Code 的 `/undo` —— 真正的后悔药

你有没有遇到过这种情况：让 Claude Code "帮我重构一下这个模块"，它一顿操作，结果代码跑不起来了，而且改的地方太多，手动回退都费劲。

**解法：`/undo`**

在 Claude Code 的对话里直接输入 `/undo`，它会**撤销上一次的全部修改**，文件状态回到你发那条指令之前。

**更精细的控制：**
- `/undo` → 撤销最近一轮操作
- 连续输入 `/undo` → 继续往前回退，像时间机器一样
- 它不只是恢复文件内容，还会把添加的新文件删掉、删掉的文件恢复回来

**💡 使用场景：**
- "重构一下" → 结果一团糟 → `/undo` → 回到解放前
- "试试另一种方案" → 不如原来的 → `/undo` → 当作没发生
- 甚至可以让 Claude 自己执行：`> 刚才的改动不太好，undo 掉` —— 它听得懂

> ⚠️ 注意：`/undo` 只能撤销 Claude 自己做的改动，你在它干活期间手动改的文件，它不一定能完美还原。所以……

## 技巧二：让 Agent 前先 `git commit` —— 终极安全网

`/undo` 很强，但**Git 才是真理**。

**养成这个习惯：在让 Agent 大规模改代码前，先切个分支或打个提交。**

```bash
# 方案 A：开分支（推荐）
git checkout -b agent-experiment

# 方案 B：快速 commit
git add .
git commit -m "checkpoint: before agent refactoring"
```

然后放心让 Agent 折腾：
- 改好了？`git diff` 看看，满意就合并
- 改崩了？`git checkout .` 或 `git reset --hard` 一键回家
- 想对比？`git diff main` 清清楚楚

**进阶玩法：让 Agent 自己管理版本**

Claude Code 和 Cursor 都支持在对话里直接说：
> "先 commit 一下当前状态，然后再开始重构"

它会真的执行 `git commit`，给你留好退路再动手。**相当于让 Agent 自己系安全带。**
